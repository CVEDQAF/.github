# ☕️ CVE DQAF (Data Quality Awareness Framework)

Welcome to the official home of the **CVE DQAF** (pronounced *"Decaf"*). 

The CVE program is entering its "data quality" era. As vulnerability data becomes more critical for automated defense, the industry needs a standardized way to define, measure, and improve the quality of the records we all rely on. This framework provides the tools to move from "just data" to **actionable intelligence**.

---

## 🎯 Our Mission
To provide CNAs, NVD, vendors, and security consumers with a robust framework for assessing the completeness, consistency, and operational utility of CVE records.

## 🚀 VulnCon 2026: "CVE Decaf"
We are officially launching the framework at [VulnCon 2026](https://www.first.org/conference/vulncon26/program#pCVE-Decaf-Brewing-Better-and-More-Actionable-Data-Quality).

* **Talk:** *CVE Decaf: Brewing Better and More Actionable Data Quality*
* **Speakers:** Jerry Gamblin (Cisco) & Jay Jacobs (Empirical Security)
* **Core Concepts:** * Defining information-quality research for the vulnerability ecosystem.
    * Establishing metrics for "actionable" data.
    * Tools for CNAs to improve their "brewing" process before publication.
